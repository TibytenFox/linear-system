# Лабораторная работа №1 по линейной алгебре: Сравнение методов решения СЛАУ
## Краткий отчёт
### 1. Постановка задачи
Разработать программу на языке C для решения систем линейных алгебраических уравнений (СЛАУ) вида \( Ax = b \), где \( A \) – квадратная невырожденная матрица, \( b \) – вектор правой части.
Требуется реализовать три метода:
- метод Гаусса (прямой ход без выбора главного элемента);
- метод Гаусса с выбором главного элемента по столбцу (частичный выбор);
- метод LU‑разложения (разложение матрицы на нижнюю треугольную \( L \) и верхнюю треугольную \( U \)) с последующим решением двух треугольных систем.
Программа должна позволять:
- ввод системы с клавиатуры и решение выбранным методом;
- автоматическое тестирование на случайных матрицах с замером времени;
- проверку точности на матрице Гильберта (оценка относительной погрешности и невязки).
---
### 2. Краткое описание реализованных алгоритмов
#### Метод Гаусса (без выбора главного элемента)
Прямой ход: матрица \( A \) и вектор \( b \) приводятся к верхнему треугольному виду путём исключения неизвестных. На \(i\)-м шаге для каждой строки \(j > i\) вычисляется множитель \( a_{ji} / a_{ii} \), затем из \(j\)-й строки вычитается \(i\)-я, умноженная на этот множитель. После прямого хода выполняется обратный ход – последовательное вычисление неизвестных из верхнетреугольной системы.
*Особенность:* алгоритм требует ненулевых диагональных элементов (пивотов) на каждом шаге.
#### Метод Гаусса с частичным выбором главного элемента
Для повышения численной устойчивости на каждом шаге прямого хода перед исключением производится поиск максимального по модулю элемента в текущем столбце среди строк от \(i\) до \(n-1\). Найденная строка меняется местами с \(i\)-й, после чего исключение выполняется как в обычном методе Гаусса.
#### LU‑разложение
Матрица \(A\) представляется в виде произведения \(L\) и \(U\), где \(L\) – нижняя треугольная с единичной диагональю, \(U\) – верхняя треугольная. Алгоритм основан на прямом ходе метода Гаусса: элементы матрицы \(U\) вычисляются как результат преобразований, а множители исключения сохраняются в матрице \(L\) (под диагональю).
Решение системы выполняется в два этапа:
1. прямой подстановкой \( Ly = b \) находится \(y\);
2. обратной подстановкой \( Ux = y \) находится \(x\).
---
### 3. Описание структуры программы
Программа разделена на несколько модулей, каждый отвечает за свою функциональность.
#### Модули работы с данными
- **`vector.h` / `vector.c`** – реализация динамического вектора: создание, удаление, клонирование, доступ по индексу, обмен элементов, вычитание векторов, печать.
- **`matrix.h` / `matrix.c`** – реализация матрицы: инициализация, удаление, клонирование, доступ к элементам, обмен строк, умножение матрицы на вектор, печать.
#### Модули алгоритмов решения
- **`gauss.h` / `gauss.c`** – функции `GaussElimination` (обычный метод Гаусса) и `GaussEliminationWithPivoting` (с выбором главного элемента).
- **`lu.h` / `lu.c`** – структура `LU`, содержащая матрицы \(L\) и \(U\). Функция `LUDecomposition` выполняет разложение, `LUSolve` решает систему по готовому разложению, `LUDelete` освобождает память.
#### Модули генерации и тестирования
- **`generate_random.h` / `generate_random.c`** – функции для генерации случайных векторов и матриц с элементами в диапазоне \([-1, 1]\) на основе стандартного генератора `random`.
- **`test.c`** – содержит:
  - функцию `Test` – сравнение времени трёх методов для одной системы заданного размера;
  - `MultipleTests` – измерение суммарного времени для нескольких правых частей при фиксированной матрице;
  - `TestHilbert` – проверка точности на матрице Гильберта (вычисление относительной погрешности и невязки);
  - меню для выбора типа тестирования;
  - управление генератором случайных чисел (фиксация seed).
#### Модуль ручного ввода
- **`main.c`** – функция `ManualInput`, запрашивающая размер системы, элементы матрицы и вектора, а затем вызывающая выбранный метод и выводящая результат.
Программа позволяет как интерактивно решать небольшие системы, так и проводить автоматизированное тестирование производительности и точности на матрицах различного размера.
## Установка и использование
1) Скачать файлы
```sh
git clone https://github.com/TibytenFox/linear-system
cd linear-system
```
2) Собрать программу
```sh
make
```
3) Запустить программу
```sh
make run
```
4) Собрать программу для тестов
```sh
make test
```
5) Запустить программу для тестов
```sh
make testing
```
6) Очистка от файлов (опционально)
```sh
make clear
```
